# 🌟 BetterMe Quiz Builder (Система Управління Квізами)

Цей проєкт являє собою повноцінну екосистему для створення, редагування та запуску динамічних опитувальників (квізів) у стилі BetterMe. Система складається з візуального редактора (Адмінки), бекенд-компілятора та рушія для проходження квиза користувачем.

## 🚀 Основні можливості

* **Візуальний конструктор (React Flow):** Створення логіки квиза шляхом перетягування карток (вузлів) та з'єднання їх стрілочками (переходами).
* **Два режими роботи:**
    * `/admin` — Візуальний граф (Drag & Drop).
    * `/admin-simple` — Спрощений табличний редактор для швидкого внесення змін.
* **Гнучке налаштування питань:** Редагування заголовків, підзаголовків, типів вузлів (Питання, Інфо, Оффер), а також управління варіантами відповідей (кнопками) безпосередньо в інтерфейсі.
* **Розумний Компілятор:** Бекенд автоматично перетворює візуальну схему в строгий формат `journey-config.json`, який розуміє рушій квиза.
* **Система Нагород (Ачівки):** Інтегрована підтримка "Системних ключів" (наприклад, `goal`, `stress`), які розблоковують нагороди на дашборді користувача.
* **Автоматичний Прогрес-бар:** Компілятор сам вираховує кроки (`Крок 1`, `Крок 2`), ігноруючи інформаційні екрани.
* **Гарантований Результат (Fallback):** Вбудована логіка запасного плану — користувач ніколи не побачить помилку "Не змогли підібрати план", навіть якщо умови не збіглися.

## 🏗 Архітектура системи

Проєкт працює за принципом розділення "Візуальної схеми" та "Бойового конфіга":

1.  **Frontend Admin (`web/src/admin`):**
    * Працює зі станом через `FlowContext.tsx`.
    * Дозволяє створювати вузли (`nodes`) та зв'язки (`edges`).
    * Зберігає поточний стан у файл бази даних адмінки ➔ `data/flow-schema.json`.
2.  **Backend Compiler (`server/src/modules/admin/admin.routes.ts`):**
    * Отримує JSON з адмінки.
    * Аналізує вузли, зв'язки та умови.
    * Генерує фінальний конфіг з масивами `nodes`, `edges`, `options`, `offers`.
    * Зберігає результат у бойовий файл ➔ `data/journey-config.json`.
3.  **Quiz Engine (`server/src/modules/session`):**
    * Читає `journey-config.json` та веде користувача по маршруту.

## ⚙️ Як працювати з Адмінкою

### 1. Створення вузлів
Натисніть "+ Питання", "+ Інфо" або "+ Оффер", щоб додати нову картку.
* **Запитання** — очікує відповіді від користувача.
* **Інфо** — інформаційний екран (прогрес-бар на ньому не збільшується).
* **Оффер** — фінальний екран результату з пропозицією підписки.

### 2. Важливі поля налаштувань
При виділенні вузла у правій панелі з'являються налаштування:
* **Системний ключ:** *Критично важливе поле.* Впишіть сюди `goal`, `stress`, `level` тощо. Це дозволяє квизу зрозуміти суть питання і видати відповідну нагороду (медаль).
* **Варіанти відповідей:** Додавайте кнопки. Поле "Текст" — те, що бачить юзер. Поле "Значення" (наприклад, `weight_loss`) — те, що використовується в логіці переходів.

### 3. Налаштування логіки (Умови переходів)
З'єднайте два вузли стрілочкою. Виділіть стрілку і в полі "Умова" пропишіть логіку у форматі `поле = значення` (наприклад, `goal = weight_loss`). Якщо поле залишити порожнім, це буде перехід "за замовчуванням".

### 4. Публікація
Після внесення змін ОБОВ'ЯЗКОВО натисніть кнопку **"💾 Опублікувати"**. Тільки після цього компілятор згенерує `journey-config.json`, і зміни з'являться в самому квизі.
